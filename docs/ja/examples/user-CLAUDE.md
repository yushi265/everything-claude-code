# ユーザーレベル CLAUDE.md の例

これはユーザーレベルの CLAUDE.md ファイルの例です。`~/.claude/CLAUDE.md` に配置してください。

ユーザーレベルの設定はすべてのプロジェクトにグローバルに適用されます。以下の用途で使用します:
- 個人的なコーディングの好み
- 常に適用したい普遍的なルール
- モジュール化されたルールへのリンク

---

## コア哲学

あなたは Claude Code です。複雑なタスクには特化したエージェントとスキルを使用します。

**主要原則:**
1. **エージェント優先**: 複雑な作業は特化したエージェントに委譲
2. **並列実行**: 可能な限り複数のエージェントで Task ツールを使用
3. **実行前に計画**: 複雑な操作には Plan Mode を使用
4. **テスト駆動**: 実装前にテストを書く
5. **セキュリティ優先**: セキュリティで妥協しない

## 個人的な好み

### プライバシー
- 常にログを編集; シークレットを貼り付けない（API キー/トークン/パスワード/JWT）
- 共有前に出力をレビュー - 機密データを削除

### コードスタイル
- コード、コメント、ドキュメントに絵文字を使用しない
- 不変性を優先 - オブジェクトや配列を変更しない
- 少数の大きなファイルよりも多数の小さなファイル
- 通常 200-400 行、最大 800 行/ファイル

### Git
- Conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- コミット前に必ずローカルでテスト
- 小さく焦点を絞ったコミット

### テスト
- TDD: テストを先に書く
- 最低 80% のカバレッジ
- 重要なフローにはユニット + インテグレーション + E2E

## 成功の指標

以下の場合に成功:
- すべてのテストが通過（80%+ カバレッジ）
- セキュリティの脆弱性がない
- コードが読みやすく保守可能
- ユーザー要件を満たしている

**哲学**: エージェント優先設計、並列実行、行動前の計画、コード前のテスト、常にセキュリティ。
