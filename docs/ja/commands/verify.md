# Verificationコマンド

現在のコードベース状態で包括的な検証を実行します。

## 手順

正確にこの順序で検証を実行します:

1. **ビルドチェック**
   - このプロジェクトのビルドコマンドを実行
   - 失敗した場合、エラーを報告して停止

2. **型チェック**
   - TypeScript/型チェッカーを実行
   - ファイル:行とともに全てのエラーを報告

3. **Lintチェック**
   - Linterを実行
   - 警告とエラーを報告

4. **テストスイート**
   - 全てのテストを実行
   - 合格/不合格数を報告
   - カバレッジパーセンテージを報告

5. **Console.log監査**
   - ソースファイル内のconsole.logを検索
   - 場所を報告

6. **Gitステータス**
   - コミットされていない変更を表示
   - 最後のコミット以降に変更されたファイルを表示

## 出力

簡潔な検証レポートを生成します:

```
VERIFICATION: [PASS/FAIL]

Build:    [OK/FAIL]
Types:    [OK/Xエラー]
Lint:     [OK/X問題]
Tests:    [X/Y合格、Z%カバレッジ]
Secrets:  [OK/X検出]
Logs:     [OK/X console.logs]

Ready for PR: [YES/NO]
```

重大な問題がある場合、修正提案とともにリスト化します。

## 引数

$ARGUMENTSは以下が可能:
- `quick` - ビルド + 型のみ
- `full` - 全チェック（デフォルト）
- `pre-commit` - コミット関連のチェック
- `pre-pr` - 全チェック + セキュリティスキャン
